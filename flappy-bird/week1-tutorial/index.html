Flappy Bird Tutorial - Part I

The Flappy Bird Game

Flappy Bird is one of the most popular game on the Internet. It is a simple one-button game in which you have to navigate our beloved bird past the obstacles. You do this by clicking your mouse repeatedly. Each time you click, the bird flies up. Here is a demonstration video for the flappy bird game.

What you will Learn:

  • Learn how to place elements, build game layouts and style elements in a HTML5 game.

    • Learn how to set, size and repeat background images using the background-image CSS property and the background-size CSS property.

  • Learn how to control elements’ layout and style dynamically through JavaScript.

  • Learn how to create animations in HTML5 games. Including:

    • Learn about how to perform and update animations using the Window.requestAnimationFrame() API.
    • How to restart animations by canceling a previous animation request using the Window.cancelAnimationFrame() API.
    • Handle animation loops and refresh frames in any HTML5 games.

Part 3 - Animating the Poles

Now we want to animate the poles so they move from left to right when we start the game. The game made it looks like the bird is flying forward, but in fact, it's the poles that are moving, our bird can only move up and down.

In order to do this, we are going to add animations to our JavaScript file.

  1. The first thing we will do is add querySelector on the elements, and initialize some variables including the bird's speed. 
xxxxxxxxxx
//index.js
// Save DOM objects to variables
const poles = document.querySelectorAll(".pole");
const gameArea = document.querySelector("#game-area");
const restartBtn = document.querySelector("#restart-btn");


const speed = 2;
let animationReq;
  1. Then we will use an addEventListener function on our restart button. When the restart button is clicked, we should set all the poles in their original position, and also start the animation for the poles. So our startGame() function will call two functions inside it, the reset()function and the gameLoop() function.
xxxxxxxxxx
//index.js
//.....
function startGame() {
    reset();
    gameLoop();
}


restartBtn.addEventListener('click', startGame);

2a. The gameLoop() Function To implement the gameLoop function, we will need to introduce the requestAnimationFrame function. The requestAnimationFrame function is slightly similar to the setInterval() function where both created a time loop. Whenever you want to update your animation on the screen, you should call the requestAnimationFrame method. The number of callbacks is usually 60 times per second (you can read more about requestAnimationFrame here). The most straightforward example of requestAnimationFrame() is:

xxxxxxxxxx
//index.js
//.....
function gameLoop() {
    //do something
    requestAnimationFrame(gameLoop);
}


gameLoop();

where the gameLoop() function keeps calling itself, then we can modify the do something line in order to make the polls move. Similar to setInterval and setTimeout, requestAnimationFrame returns an ID when calling the function, and you can use the ID to cancel the looping. 

xxxxxxxxxx


//index.js
let animationReq
function gameLoop() {
    updatePoles()
    animationReq = requestAnimationFrame(gameLoop);
}
cancelAnimationFrame(animationReq);
    

2b. UpdatePoles() function Then we want to consider how to move the poles by changing the "right" attribute in the previous "to do" section. We know that the requestAnimationFrame makes a time loop, and if we add the speed to the current position, we can move the poles every frame of the animation. So first we want to find the current pole's position by using getComputedStyle, which will return all the information about the element's CSS.

xxxxxxxxxx
//index.js
function updatePoles() {
    // Move poles
    let polesCurrentPos = parseFloat(
        window.getComputedStyle(poles[0]).getPropertyValue("right")
    );
}

Then we can add the speed for every pole by setting the "right" attribute. 

xxxxxxxxxx
//index.js
function updatePoles() {
    // Move poles
    let polesCurrentPos = parseFloat(
        window.getComputedStyle(poles[0]).getPropertyValue("right")
    );


    poles.forEach((pole) => {
        pole.style.right = `${polesCurrentPos + speed}px`;
    });
}

2c. The reset() function Inside the reset function, since querySelectorAll returns a node list, we are looping through all the poles and resetting all the values of the right attribute back to 0. We would also need to cancel the previous animation by using cancelAnimationFrame.

xxxxxxxxxx
//index.js
function reset() {
    poles.forEach((pole) => {
        pole.style.right = 0;
    });
    if (animationReq) {
        cancelAnimationFrame(animationReq);
    }
}
    

Now we can put all the index.js code together, and you should have something that looks like the code below:

xxxxxxxxxx
"use strict";


// Save DOM objects to variables
const poles = document.querySelectorAll(".pole");
const gameArea = document.querySelector("#game-area");
const containerWidth = gameArea.clientWidth;
const containerHeight = gameArea.clientHeight;
const restartBtn = document.querySelector("#restart-btn");


// make some variables accesible to functions.
const speed = 2;
let animationReq;




function startGame() {
    reset();
    gameLoop();
}


function updatePoles() {
    // Move poles
    let polesCurrentPos = parseFloat(
        window.getComputedStyle(poles[0]).getPropertyValue("right")
    );


    poles.forEach((pole) => {
        pole.style.right = `${polesCurrentPos + speed}px`;
    });
}




function gameLoop() {
    updatePoles();
    animationReq = requestAnimationFrame(gameLoop);
}


function reset() {
    poles.forEach((pole) => {
        pole.style.right = 0;
    });

        cancelAnimationFrame(animationReq);
    }
}


restartBtn.addEventListener('click', startGame);

Part4 - Re-generating Poles

In part 2, the poles disappear after moving to the left. The last thing we will do is to generate new poles by moving the pole elements to the right of the game area.The new poles should have a different height that is randomly generated, while the gap between the pair of poles are fixed

So whenever the old poles disappear to the left side of the game area, generate new pair of poles. First, we need to determine whether the poles reached the left side of the game area. Since polesCurrentPos (used in the previous part) is the length between the rightmost game area and the pole, that means if polesCurrentPos is bigger than the game area's width, containerWidth, the poles are moving out of the game area. 

xxxxxxxxxx
//index.js
const containerWidth = gameArea.clientWidth;


function updatePoles() {
    // Move poles
    let polesCurrentPos = parseFloat(
        window.getComputedStyle(poles[0]).getPropertyValue("right")
    );
    //  Check whether the poles went putside of game area.
    if (polesCurrentPos > containerWidth) {
        // Generate new poles here  
    }
}

Then we will generate a random height using Math. random and set the two pole's heights accordingly. Then we need to set the new pole's right attribute to 0 in order to place it on the right side of the game area. 

xxxxxxxxxx
//index.js
if (polesCurrentPos > containerWidth) {
    // Generate new poles.
    let newHeight = parseInt(Math.random() * 100);
    // Change the poles' height
    //The middle gap of the pole is 100px
    poles[0].style.height = `${100 + newHeight}px`;
    poles[1].style.height = `${100 - newHeight}px`;


    // Move poles back to the right-hand side of game area.
    polesCurrentPos = 0; // This is based on the "right" property.
}
